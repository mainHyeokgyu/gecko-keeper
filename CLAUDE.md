## 📋 아이디어 빌딩 규칙

### ✅ 논의할 것
- **프로토타입 구현**에 직접 필요한 기술적 결정사항
- **MVP 범위** 내의 구체적 기능 정의
- **즉시 실행 가능**한 구현 방안
- **측정 가능한** 성능/정확도 기준
- **명확한 우선순위**가 있는 기능 선택

### ❌ 논의하지 않을 것  
- 사업화 전략 (수익모델, 시장분석, 경쟁사 등)
- 확장성 논의 (다종 대응, B2B 등)
- 추상적이고 모호한 아이디어
- MVP 범위를 벗어나는 고급 기능
- 구현 불가능한 이상적인 시나리오

### 🔧 논의 방식
1. **구체적 수치** 기반 결정 (온도 ±0.5도, 습도 60% 등)
2. **바이너리 선택** 우선 (A vs B, 포함 vs 제외)
3. **즉시 구현 가능** 여부 확인
4. **1주 내 프로토타입** 목표로 범위 제한

### 🏷️ 현재 포커스
- **센서**: 온습도 정확도, 설치방법
- **분무기**: 제어방식, 작동시간  
- **통신**: MQTT vs HTTP, 데이터 형식
- **UI**: 핵심 3개 기능만 (상태/제어/로그)
